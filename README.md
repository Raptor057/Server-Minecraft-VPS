# Crafty Controller for Minecraft

ES: Stack Docker Compose para levantar `Crafty Controller` y administrar servidores de Minecraft desde una UI web. Queda listo para publicar en GitHub sin exponer configuracion privada ni mundos.

EN: Docker Compose stack to run `Crafty Controller` and manage Minecraft servers from a web UI. It is ready to be published on GitHub without exposing private configuration or world data.

## Includes / Incluye

- `Crafty Controller` in Docker / `Crafty Controller` en Docker
- Persistent volumes for config, logs, backups, imports, and servers / Volumenes persistentes para config, logs, backups, imports y servidores
- Environment variables in `.env.example` / Variables de entorno en `.env.example`
- Setup ready to create `Minecraft Bedrock` or `Minecraft Java` servers from the UI / Configuracion lista para crear servidores `Minecraft Bedrock` o `Minecraft Java` desde la UI

## Requirements / Requisitos

- Docker
- Docker Compose
- A Linux VPS / Un VPS Linux

## Quick Start on Linux / Inicio rapido en Linux

1. Clone the repo and enter the folder. / Clona el repo y entra a la carpeta.

```bash
git clone <YOUR_REPO>
cd Server-Minecraft-VPS
```

2. Create your local environment file. / Crea tu archivo local de entorno.

```bash
cp .env.example .env
```

3. Start Crafty. / Levanta Crafty.

```bash
docker compose up -d
```

4. Open the panel. / Abre el panel.

```text
https://YOUR_IP:8443
```

Crafty uses HTTPS with a self-signed certificate by default, so your browser will show a warning. This is expected.

Crafty usa HTTPS con certificado autofirmado por defecto, asi que el navegador mostrara una advertencia. Es normal.

5. Read the initial credentials. / Saca las credenciales iniciales.

```bash
cat ./crafty/config/default-creds.txt
```

6. Log into the panel and create your server. / Entra al panel y crea tu servidor.

- Choose `Minecraft-Bedrock` or `Minecraft-Java` / Elige `Minecraft-Bedrock` o `Minecraft-Java`
- For Bedrock use port `19132` / Para Bedrock usa el puerto `19132`
- For Java use a port inside `25500-25600` / Para Java usa un puerto dentro del rango `25500-25600`

## Structure / Estructura

- `docker-compose.yml`: main Crafty service / servicio principal de Crafty
- `.env.example`: public configuration template / plantilla publica de configuracion
- `crafty/config`: database, certificates, and initial credentials / base de datos, certificados y credenciales iniciales
- `crafty/servers`: servers created from the UI / servidores creados desde la UI
- `crafty/backups`: backups generated by Crafty / backups generados por Crafty
- `crafty/import`: directory to import existing servers / directorio para importar servidores existentes

## Important Variables / Variables importantes

| Variable | English | Espanol | Default |
| --- | --- | --- | --- |
| `CRAFTY_PANEL_PORT` | Crafty HTTPS panel port | Puerto HTTPS del panel de Crafty | `8443` |
| `CRAFTY_DYNMAP_PORT` | Dynmap port if used | Puerto de Dynmap si lo usas | `8123` |
| `CRAFTY_BEDROCK_PORT` | Bedrock UDP port | Puerto UDP de Bedrock | `19132` |
| `CRAFTY_JAVA_PORT_RANGE` | TCP port range for Java servers | Rango TCP para servidores Java | `25500-25600` |
| `TZ` | Container timezone | Zona horaria del contenedor | `America/Chicago` |

## VPS Firewall / Firewall del VPS

Open these ports in your VPS firewall or cloud provider firewall:

Abre estos puertos en el firewall del VPS o del proveedor:

- `8443/tcp`: Crafty web panel / panel web de Crafty
- `19132/udp`: Minecraft Bedrock server / servidor Minecraft Bedrock
- `8123/tcp`: Dynmap, only if you use it / Dynmap, solo si lo vas a usar
- `25500-25600/tcp`: range for Java servers created from Crafty / rango para servidores Java creados desde Crafty

### UFW

```bash
sudo ufw allow 8443/tcp
sudo ufw allow 19132/udp
sudo ufw allow 8123/tcp
sudo ufw allow 25500:25600/tcp
sudo ufw reload
```

### Firewalld

```bash
sudo firewall-cmd --permanent --add-port=8443/tcp
sudo firewall-cmd --permanent --add-port=19132/udp
sudo firewall-cmd --permanent --add-port=8123/tcp
sudo firewall-cmd --permanent --add-port=25500-25600/tcp
sudo firewall-cmd --reload
```

If you only plan to use Bedrock, `8443/tcp` and `19132/udp` are enough.

Si solo vas a usar Bedrock, con `8443/tcp` y `19132/udp` basta.

## Safe Publishing / Publicacion segura

Before pushing changes to GitHub:

Antes de subir cambios a GitHub:

- do not commit `.env` / no subas `.env`
- do not commit `crafty/` / no subas `crafty/`
- do not commit worlds, backups, or generated credentials / no subas mundos, backups ni credenciales generadas por Crafty
